get_drag(state_inp: ndarray, Am_ratio=0, qm_ratio=0, CD=0, CR=0, jd=2459714.2916666665, r_obj_m=0, order=20)#

Calculates the drag acceleration vector between the satellite and the Earth’s atmosphere based on the state vector and various ratios and constants.

Parameters:

  • state_inp (numpy.ndarray) – The state vector representing the satellite’s position and velocity.

  • Am_ratio (float) – The ratio of the satellite’s mass to the mass of the Earth.

  • qm_ratio (float, optional) – The ratio of the satellite’s mass to the mass of the Moon. Defaults to 0.

  • CD (float, optional) – The drag coefficient. Defaults to 2.2.

  • CR (float, optional) – The reflection coefficient. Defaults to 1.25.

  • jd (float, optional) – The Julian date. Defaults to the current Julian date.

  • r_obj_m (float, optional) – The radius of the object in meters. Defaults to 20e-6.

Returns:

The drag acceleration vector.

Return type:

numpy.ndarray

Description:

This function calculates the drag acceleration vector between the satellite and the Earth’s atmosphere based on the state vector and various ratios and constants. It uses the satellite’s position and velocity to calculate the drag acceleration.

The drag acceleration is calculated using the formula:

a_drag = -0.5 * CD * Am_ratio * rho * v_rel^2 * v_rel_vec / v_rel

where:

  • CD is the drag coefficient

  • Am_ratio is the ratio of the satellite’s mass to the mass of the Earth

  • rho is the density of the atmosphere at the satellite’s position

  • v_rel is the relative velocity between the satellite and the atmosphere

  • v_rel_vec is the vector of the relative velocity between the satellite and the atmosphere

The function returns the drag acceleration vector.

get_earth_rad(state_inp: ndarray, Am_ratio=0, qm_ratio=0, CD=0, CR=0, jd=2459714.2916666665, r_obj_m=0, order=20)#

Calculate the Earth radiation pressure effect on a satellite.

Parameters:

  • state_inp (numpy.ndarray) – The state vector of the satellite in ECI coordinates.

  • Am_ratio (float) – The area-to-mass ratio of the satellite.

  • qm_ratio (float, optional) – The charge-to-mass ratio of the satellite. Defaults to 0.

  • CD (float, optional) – The drag coefficient of the satellite. Defaults to 2.2.

  • CR (float, optional) – The reflection law effectiveness coefficient. Defaults to 1.25.

  • jd (float, optional) – The Julian date. Defaults to NOW_JD.

  • r_obj_m (float, optional) – The radius of the satellite in meters. Defaults to 20e-6.

Returns:

The acceleration due to Earth radiation pressure in ECI coordinates.

Return type:

numpy.ndarray

Notes

  • The Earth radiation pressure is calculated using the method described in “EARTH RADIATION PRESSURE EFFECTS ON SATELLITES” by Philip Knoche.

  • The function assumes that the satellite is a perfect reflector and emits equally in all directions.

  • The function uses the solar position and the satellite’s position in ECI coordinates to calculate the solar zenith angle and the solar irradiance.

  • The function uses the area-to-mass ratio, charge-to-mass ratio, drag coefficient, reflection law effectiveness coefficient, and the satellite’s position in ECI coordinates to calculate the Earth radiation pressure acceleration.

get_tides(state_inp: ndarray, Am_ratio=0, qm_ratio=0, CD=0, CR=0, jd=2459714.2916666665, r_obj_m=0, order=20)#

Calculates the tidal acceleration vector based on the state vector and various ratios and constants.

Parameters:

  • state_inp (numpy.ndarray) – The state vector representing the satellite’s position and velocity.

  • Am_ratio (float, optional) – The ratio of the satellite’s mass to the mass of the Earth. Defaults to 0.

  • qm_ratio (float, optional) – The ratio of the satellite’s mass to the mass of the Moon. Defaults to 0.

  • CD (float, optional) – The drag coefficient. Defaults to 2.2.

  • CR (float, optional) – The reflection coefficient. Defaults to 1.25.

  • jd (float, optional) – The Julian date. Defaults to the current Julian date.

  • r_obj_m (float, optional) – The radius of the object in meters. Defaults to 20e-6.

Returns:

The tidal acceleration vector.

Return type:

numpy.ndarray

Description:

This function calculates the tidal acceleration vector based on the state vector and various ratios and constants. It uses the positions of the Sun and the Moon to calculate the tidal acceleration.

The tidal acceleration is calculated using the formula:

a_tides = (a_sun + a_moon) * r / r_norm

where:

  • a_sun is the tidal acceleration due to the Sun

  • a_moon is the tidal acceleration due to the Moon

  • r is the vector from the satellite to the center of the Earth

  • r_norm is the norm of the vector r

The values of a_sun and a_moon are calculated using the formula:

a_sun = -3/2 * kap * MU_SUN * RE5 * (3 * cosgam2 - 1) / (s_norm**3 * rnorm4) a_moon = -3/2 * kap * MU_MOON * RE5 * (3 * cosgam2 - 1) / (m_norm**3 * rnorm4)

where:

  • kap is a constant value of 0.3

  • MU_SUN is the gravitational constant of the Sun

  • MU_MOON is the gravitational constant of the Moon

  • RE is the Earth’s radius

  • s_norm is the norm of the vector from the satellite to the Sun

  • m_norm is the norm of the vector from the satellite to the Moon

  • rnorm4 is the fourth power of the norm of the vector r

The function returns the tidal acceleration vector.

sphericals(state_inp: ndarray, Am_ratio=0, qm_ratio=0, CD=0, CR=0, jd=2459714.2916666665, r_obj_m=0, order=20)#

Calculates the gravitational acceleration vector of a satellite due to the gravitational field of the Earth.

Parameters:

  • state_inp (numpy.ndarray) – The state vector of the satellite.

  • Am_ratio (float, optional) – The area-to-mass ratio of the satellite. Defaults to 0.

  • qm_ratio (float, optional) – The charge-to-mass ratio of the satellite. Defaults to 0.

  • CD (float, optional) – The drag coefficient of the satellite. Defaults to 2.2.

  • CR (float, optional) – The reflectivity coefficient of the satellite. Defaults to 1.25.

  • jd (float, optional) – The Julian date. Defaults to NOW_JD.

  • r_obj_m (float, optional) – The radius of the object in meters. Defaults to 20e-6.

  • order (int, optional) – The order of the spherical harmonics expansion. Defaults to 7.

Returns:

The gravitational acceleration vector of the satellite.

Return type:

numpy.ndarray

Raises:

ValueError – If the shape of the input state vector is not (6,).

Notes

  • The function uses the spherical harmonics expansion of the gravitational potential to calculate the gravitational acceleration vector.

  • The order of the spherical harmonics expansion is controlled by the order parameter.

  • The function assumes that the input state vector is in ECI coordinates.

  • The function calculates the gravitational acceleration vector in ECI coordinates.

  • The function assumes that the input state vector is in meters.

  • The function assumes that the input Julian date is in UTC.

  • The function assumes that the input radius of the object is in meters.
