src_dust.sim_funcs._gaussint(P, x, y, r, terms=3, log=False)#

Compute the Gaussian integral with optional higher-order terms.

Parameters:

  • P (numpy.ndarray) – A 2x2 positive-definite matrix.

  • x (float) – x-coordinate of the point.

  • y (float) – y-coordinate of the point.

  • r (float) – Radius of the satellite.

  • terms (int, optional) – Number of terms to include in the expansion (default is 3).

  • log (bool, optional) – If True, return the logarithm of the result (default is False).

Returns:

The computed Gaussian integral value (or its logarithm if log=True).

Return type:

float

Notes

  • The function computes the Gaussian integral using the inverse of matrix P.

  • Higher-order terms are included based on the value of terms.

  • If log is True, the logarithm of the result is returned.

class src_dust.sim_funcs.DustHit_NewStrategy(dust_means_inp, dust_covars_inp, sat_mean, sat_covar, v_unit_sat, r_sat, ws, plot=False)#

Bases: object

Class for calculating the probability of a dust grain hitting a satellite.

This class computes the probability of collision by projecting dust states and the satellite state onto a common subspace and then evaluating the convolution of the corresponding Gaussian mixtures.

Parameters:

  • dust_means_inp (numpy.ndarray) – Array of dust mean vectors (at least the first three columns represent position) in km.

  • dust_covars_inp (numpy.ndarray) – Array of dust covariance matrices (first three rows/columns correspond to position).

  • sat_mean (numpy.ndarray) – Mean state vector of the satellite (first three elements represent position) in km.

  • sat_covar (numpy.ndarray) – Covariance matrix of the satellite (first three rows/columns correspond to position).

  • v_unit_sat (numpy.ndarray) – Unit vector of the satellite’s velocity.

  • r_sat (float) – Radius of the satellite (in m).

  • ws (numpy.ndarray) – Weights for the dust particles.

  • plot (bool, optional) – Whether to plot the simulation (default is False).

get_convdist_func(num_distributions, max_P_val, m_grain, m_desired, p_desired)#

Compute the convolution distribution function for the dust-satellite collision model.

This method projects the dust means and covariances relative to the satellite, generates a set of angles spanning [0, 2π), and for each angle computes the crossing vectors and corresponding convolution values.

Parameters:

  • num_distributions (int) – The number of minor distributions (components) in the dust model.

  • max_P_val (float) – The maximum probability value used as a threshold in the crossing search.

  • m_grain (float) – The mean mass of a dust grain.

  • m_desired (float) – The desired total mass.

  • p_desired (float) – The desired probability threshold.

Returns:

A tuple containing: - Array of angles (in radians) used for the search. - List of crossing vectors for each theta. - List of convolution probability values computed at each crossing. - List of mass values computed for each crossing.

Return type:

tuple

static get_crossings(theta, dust_means, dust_covars, ws, sat_covar, r_sat, max_P_val, m_grain, m_desired, p_desired=0.95, num_chi=100)#

Calculate the crossing points of a vector (rotated by an angle theta) with the Gaussian mixture model (GMM) representing dust particles.

The method rotates the satellite covariance’s Cholesky factor by theta to define a search direction. It then uses a recursive bracketing approach (via Brent’s method) to find all values of the scalar chi for which the difference between the computed convolution value and max_P_val changes sign.

Parameters:

  • theta (float) – The angle by which to rotate the search direction (in radians).

  • dust_means (numpy.ndarray) – Mean vectors for the dust components (projected).

  • dust_covars (numpy.ndarray) – Covariance matrices for the dust components (projected).

  • ws (numpy.ndarray) – Weights for the dust components.

  • sat_covar (numpy.ndarray) – Satellite covariance matrix (projected).

  • r_sat (float) – Radius of the satellite (in km).

  • max_P_val (float) – Maximum probability value for the crossing condition.

  • m_grain (float) – Grain mass parameter.

  • m_desired (float) – Desired mass parameter.

  • p_desired (float, optional) – Desired probability threshold (default is 0.95).

  • num_chi (int, optional) – Number of chi values to use in the search (default is 100).

Returns:

A tuple containing: - A list of vectors representing the crossing points. - A list of mass values computed for each crossing.

Return type:

tuple

  1. _get_conv_gmm_value

src_dust.sim_funcs._get_conv_gmm_value(means, covars, ws, r_sat, pos)#

Compute the convolution value for a Gaussian Mixture Model (GMM) with a satellite.

Parameters:

  • means (list of numpy.ndarray) – List of mean vectors for each Gaussian component.

  • covars (list of numpy.ndarray) – List of covariance matrices for each Gaussian component.

  • ws (list of float) – List of weights for each Gaussian component.

  • r_sat (float) – Radius of the satellite.

  • pos (numpy.ndarray) – Position vector of the point at which to evaluate the integral.

Returns:

The computed convolution value for the GMM with the satellite.

Return type:

float

Notes

  • The function iterates over each Gaussian component in the GMM.

  • For each component, it computes the Gaussian integral using the _gaussint function.

  • The result is weighted by the component’s weight and summed to get the final value.
